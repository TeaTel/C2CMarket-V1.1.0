# 前端页面重构活动方案文档

---

## 1. 项目背景与目标

### 1.1 项目背景

现有"校园集市"平台已实现基础的商品交易、社区帖子、消息聊天、圈子浏览等核心功能。当前前端采用 Vue 3 + Vue Router + Axios 技术栈，已具备基础的响应式设计和组件化架构。但随着平台内容日益丰富，当前首页结构（集市/社区双Tab切换）已无法满足多维度内容消费需求，底部导航栏的6按钮布局也显得拥挤、信息架构不够清晰。

### 1.2 重构目标

基于现有项目内容，对前端页面进行交互功能重构，优化用户体验，实现以下核心目标：

- **多维度内容展示**：首页支持"发现"、"关注"、"校区"三种内容分类维度的快速切换
- **快速导航筛选**：通过圈子副菜单栏实现内容的快捷筛选
- **底部导航简化**：优化为5按钮布局，新增"活动"入口，突出"发布"操作
- **全平台兼容**：确保在PC端、平板及移动设备上的正常显示与操作

## 2. 核心功能重构要求

### 2.1 首页内容与访问控制

#### 2.1.1 访问权限

- 首页（`/` 路径）面向**所有用户**开放浏览权限，无需登录即可访问
- 未登录用户可浏览"发现"和"校区"分类内容
- "关注"分类内容需要登录后才能查看（未登录时引导用户登录）
- 保持现有路由白名单机制，`/` 路径已在 `publicPaths` 中

#### 2.1.2 内容展示规则

首页主要内容区域展示其他用户上传的帖子和交易信息，以信息流（Feed）形式呈现。

**三种内容分类展示机制：**

| 分类 | 数据来源 | 展示规则 |
|------|----------|----------|
| **发现**（默认） | 平台全局内容池 | 展示所有用户上传的帖子和交易，按时间倒序/热度排序，支持分页加载 |
| **关注** | 当前用户关注账号的发布 | 仅显示已关注用户发布的帖子和交易，需登录后可见；无关注内容时显示引导提示 |
| **校区** | 同校区用户的发布 | 根据当前用户账户设置中指定的校区，仅显示同校区用户上传的内容；未设置校区时提示用户设置 |

#### 2.1.3 数据流设计

```
┌─────────────────────────────────────────────────────┐
│                    首页 Feed 组件                      │
├─────────────────────────────────────────────────────┤
│  activeTab: 'discover' | 'following' | 'campus'     │
│  currentPage / pageSize / hasMore / loading          │
│                                                       │
│  根据 activeTab 调用不同API:                           │
│  ┌──────────────┬─────────────────────────────────┐ │
│  │ discover     │ GET /api/v2/feed?type=discover  │ │
│  │ following    │ GET /api/v2/feed?type=following │ │
│  │ campus       │ GET /api/v2/feed?type=campus    │ │
│  └──────────────┴─────────────────────────────────┘ │
│                                                       │
│  FeedItem 统一数据结构:                                 │
│  { id, itemType: 'POST'|'PRODUCT', title, content,   │
│    coverImage, price, userAvatar, userName,           │
│    createdAt, likeCount, commentCount, viewCount }    │
└─────────────────────────────────────────────────────┘
```

> **复用说明**：现有 `feedApi.getFeed(params)` 已支持分页参数 `{ page, size }`，需要后端扩展支持 `type` 参数区分内容分类。

#### 2.1.4 交互状态说明

| 状态 | 表现 | 处理方式 |
|------|------|----------|
| **加载中** | 骨架屏占位（4-6个卡片骨架） | 复用现有 `.skeleton-card` 样式 |
| **加载成功（有数据）** | 正常展示Feed列表 | 复用 `PostCard` 组件 + Product卡片 |
| **加载成功（无数据）** | 展示空状态占位图 + 引导文案 | 根据tab展示不同引导语 |
| **加载失败** | 展示错误提示 + "重试"按钮 | 复用现有 `.error-state` 组件 |
| **未登录访问"关注"** | 展示登录引导卡片 + "去登录"按钮 | 点击跳转登录页（带redirect参数） |
| **滚动加载更多** | 底部"加载中..."指示器 | 复用现有 `handleScroll` + `loadMore` 逻辑 |

### 2.2 顶部主菜单栏设计

#### 2.2.1 布局结构

```
┌──────────────────────────────────────────────────────────┐
│  [ 关注 ]    [ 发现 ]    [ 校区 ]              🔍 ______ │
│   (未激活)   (默认激活)   (未激活)         (搜索关键词)    │
└──────────────────────────────────────────────────────────┘
```

#### 2.2.2 按钮规格

- **三个导航按钮**：关注、发现、校区，等宽分布
- **激活态样式**：
  - 文字颜色：`#FF6A00`（品牌橙色）
  - 底部指示条：3px 实线，颜色 `#FF6A00`
  - 字体加粗：`font-weight: 700`
- **非激活态样式**：
  - 文字颜色：`#666`
  - 字体常规：`font-weight: 500`
- **点击切换动画**：底部指示条 `transition: left 0.25s cubic-bezier(0.4, 0, 0.2, 1)` 平滑滑动
- **搜索框**：位于菜单栏最右侧，样式与现有搜索框保持一致（圆角胶囊型，宽度约120-160px，聚焦时展开）

#### 2.2.3 交互流程

```
用户点击 Tab → activeTab 响应式更新 → Feed 数据重新拉取
                                        ↓
                        显示对应分类的帖子和交易列表
                                        ↓
                        重置分页参数 (page=1)
                        滚动位置回到顶部
```

### 2.3 副菜单栏实现

#### 2.3.1 布局结构

副菜单栏位于主菜单栏下方，作为圈子内容的快速索引：

```
┌──────────────────────────────────────────────────────────┐
│  [全部] [数码] [书籍] [生活] [服饰] [美妆] [运动] [其他] →  │
│   (可横向滚动查看更多圈子)                                   │
└──────────────────────────────────────────────────────────┘
```

#### 2.3.2 功能说明

- 展示各圈子名称和图标，支持**横向滚动**（`overflow-x: auto` + `scroll-snap-type: x mandatory`）
- 每个圈子标签为可点击按钮，点击后切换当前选中的圈子，主内容区域仅显示该圈子下的内容
- "全部"选项为默认选中状态，展示所有圈子的内容
- 圈子数据从后端API动态获取：`GET /api/v2/categories` 或现有 `categoryApi.getCategories()`
- 支持分页或虚拟滚动，当圈子数量较多时保证性能

#### 2.3.3 交互流程

```
用户点击圈子标签 → activeCircle 更新
                        ↓
        主内容区域 Feed 数据重新拉取（带 circleId 参数）
                        ↓
        GET /api/v2/feed?type=discover&circleId={id}&page=1
                        ↓
        滚动位置回到顶部
```

### 2.4 底部菜单栏功能

#### 2.4.1 布局结构

```
┌──────────┬──────────┬──────────┬──────────┬──────────┐
│   首页   │   消息   │    ⊕     │   活动   │   我     │
│   🏠    │   💬    │  (突出)  │   📅    │   👤    │
│  (home)  │(message) │ (publish)│(activity)│(profile) │
└──────────┴──────────┴──────────┴──────────┴──────────┘
```

#### 2.4.2 五按钮定义

| 序号 | 按钮名称 | 图标 | 路由目标 | 功能描述 | 认证要求 |
|------|----------|------|----------|----------|----------|
| 1 | **首页** | 🏠 (home SVG) | `/` | 返回首页，展示聚合内容Feed | 无需登录 |
| 2 | **消息** | 💬 (message SVG) | `/messages` | 进入消息页面，查看和回复私聊消息；显示未读消息数徽章 | 需要登录 |
| 3 | **发布** | ⊕ (突出圆形按钮) | 弹出发布选择面板 | 打开发布界面，支持选择"发布帖子"或"发布交易" | 需要登录 |
| 4 | **活动** | 📅 (calendar SVG) | `/activities` | 进入活动页面，展示学校社团或组织发布的活动信息 | 无需登录 |
| 5 | **我** | 👤 (profile SVG) | `/profile` | 进入个人中心，展示账号信息、发布列表、收藏、订单等 | 需要登录 |

#### 2.4.3 "发布"按钮特殊设计

- 使用醒目的 **"+"** 符号（而非文字标签），圆形橙色渐变背景
- 按钮整体向上偏移突起（`margin-top: -16px`），视觉上突出
- 尺寸：44px × 44px（图标区域），比其余图标大
- 背景：`linear-gradient(135deg, #FF6A00 0%, #FF8533 100%)`
- 阴影：`0 4px 12px rgba(255, 106, 0, 0.35)`
- 点击时非直接跳转，而是弹出底部动作面板（ActionSheet），提供两个选项：
  - "发布帖子" → 跳转 `/community/posts/create`
  - "发布交易" → 跳转 `/products/create`

#### 2.4.4 现有NavBar改造对照

| 现有按钮（6个） | 重构后按钮（5个） | 说明 |
|:---:|:---:|---|
| 首页 | 首页 ✓ | 保留，功能不变 |
| 社区 | — | 移除，社区内容整合到首页"发现"中 |
| 圈子 | — | 移除，圈子作为首页副菜单栏 |
| 发布(+) | 发布(+) ✓ | 保留，增加发布类型选择面板 |
| 消息 | 消息 ✓ | 保留，功能不变 |
| 我的 | 我 ✓ | 保留，功能不变 |
| — | **活动** 🆕 | 新增，需新建页面 |

**需要新增的路由：**

#### 4.3.2 活动API（新增）

```javascript
export const activityApi = {
  // 获取活动列表
  getActivities(params) {
    return api.get('/v2/activities', { params })
    // params: { page, size, status: 'upcoming'|'ongoing'|'past', campusId }
  },
  
  // 获取活动详情
  getActivityDetail(id) {
    return api.get(`/v2/activities/${id}`)
  },
  
  // 报名参加活动
  joinActivity(id) {
    return api.post(`/v2/activities/${id}/join`)
  },
  
  // 取消报名
  cancelJoin(id) {
    return api.delete(`/v2/activities/${id}/join`)
  }
}
```

### 4.5 浏览器兼容性

- 使用 **CSS Grid** 和 **Flexbox** 布局（主流浏览器均支持）
- SVG 图标内联（避免额外请求，所有浏览器原生支持）
- 使用 `env(safe-area-inset-bottom)` 适配 iPhone X+ 刘海屏
- CSS过渡使用 `cubic-bezier` 函数（标准CSS，全兼容）
- 滚动行为使用 `-webkit-overflow-scrolling: touch` 优化 iOS 滚动体验
- 使用 `scrollbar-width: none` (Firefox) 和 `::-webkit-scrollbar` (Chrome/Safari) 隐藏副菜单滚动条
