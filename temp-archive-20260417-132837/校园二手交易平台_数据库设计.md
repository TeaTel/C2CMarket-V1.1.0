## 表结构设计

### 1. users 表（用户表）

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | NOT NULL UNIQUE | 用户名 |
| password_hash | VARCHAR(255) | NOT NULL | 密码哈希值 |
| phone | VARCHAR(20) | NOT NULL UNIQUE | 手机号 |
| wechat | VARCHAR(50) | UNIQUE | 微信号 |
| qq | VARCHAR(20) | UNIQUE | QQ号 |
| avatar | VARCHAR(255) | | 头像URL |
| school | VARCHAR(100) | | 学校 |
| major | VARCHAR(100) | | 专业 |
| is_student | BOOLEAN | DEFAULT FALSE | 是否学生身份 |
| status | ENUM('active', 'banned') | DEFAULT 'active' | 用户状态 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**设计理由：**
- `username`、`phone` 设为唯一，确保用户标识的唯一性
- `password_hash` 存储加密后的密码，保障安全
- `status` 字段用于管理员管理用户状态，支持封禁功能
- 增加 `is_student` 字段支持学生身份验证需求
- 时间戳字段用于记录用户注册和更新时间

**索引设计：**
- 主键 `id` 自动创建索引
- `username`、`phone` 创建唯一索引，加速查询
- `status` 创建普通索引，便于管理员筛选用户

### 2. categories 表（分类表）

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 分类ID |
| name | VARCHAR(50) | NOT NULL UNIQUE | 分类名称 |
| parent_id | INT | REFERENCES categories(id) | 父分类ID，支持多级分类 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**设计理由：**
- 支持多级分类结构，通过 `parent_id` 实现
- `name` 设为唯一，确保分类名称不重复
- 符合用户故事中商品分类的需求

**索引设计：**
- 主键 `id` 自动创建索引
- `name` 创建唯一索引，加速分类查找
- `parent_id` 创建普通索引，便于查询子分类

### 3. products 表（商品表）

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 商品ID |
| name | VARCHAR(100) | NOT NULL | 商品名称 |
| description | TEXT | | 商品描述 |
| price | DECIMAL(10,2) | NOT NULL | 商品价格 |
| status | ENUM('available', 'sold', 'removed') | DEFAULT 'available' | 商品状态 |
| category_id | INT | REFERENCES categories(id) | 分类ID |
| user_id | INT | REFERENCES users(id) | 卖家ID |
| view_count | INT | DEFAULT 0 | 浏览量 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**设计理由：**
- `status` 字段跟踪商品状态，支持上架、已售出、已下架等状态
- `view_count` 用于统计商品浏览量，符合用户故事需求
- 与 `categories` 和 `users` 表建立外键关系
- 时间戳字段用于排序和管理

**索引设计：**
- 主键 `id` 自动创建索引
- `user_id` 创建普通索引，加速查询用户的商品
- `category_id` 创建普通索引，加速按分类查询
- `status` 创建普通索引，便于筛选商品状态
- `created_at` 创建普通索引，支持按时间排序

### 4. images 表（商品图片表）

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 图片ID |
| product_id | INT | REFERENCES products(id) | 商品ID |
| url | VARCHAR(255) | NOT NULL | 图片URL |
| order_index | INT | DEFAULT 0 | 图片顺序 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**设计理由：**
- 单独存储商品图片，支持多图上传（最多9张）
- `order_index` 用于控制图片显示顺序
- 与 `products` 表建立外键关系

**索引设计：**
- 主键 `id` 自动创建索引
- `product_id` 创建普通索引，加速查询商品的所有图片
- `order_index` 创建普通索引，确保图片按顺序显示

### 5. orders 表（订单表）

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 订单ID |
| buyer_id | INT | REFERENCES users(id) | 买家ID |
| seller_id | INT | REFERENCES users(id) | 卖家ID |
| total_amount | DECIMAL(10,2) | NOT NULL | 订单总金额 |
| status | ENUM('pending', 'paid', 'shipped', 'completed', 'cancelled', 'refunded') | DEFAULT 'pending' | 订单状态 |
| payment_method | ENUM('platform', 'offline') | | 支付方式 |
| address | VARCHAR(255) | | 收货地址 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**设计理由：**
- `status` 字段跟踪订单状态，支持完整的订单流程
- `payment_method` 支持平台担保交易和线下支付
- 同时记录买家和卖家ID，便于双方查看订单
- 时间戳字段用于跟踪订单进度

**索引设计：**
- 主键 `id` 自动创建索引
- `buyer_id` 和 `seller_id` 创建普通索引，加速查询用户的订单
- `status` 创建普通索引，便于筛选订单状态
- `created_at` 创建普通索引，支持按时间排序

### 6. order_items 表（订单项表）

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 订单项ID |
| order_id | INT | REFERENCES orders(id) | 订单ID |
| product_id | INT | REFERENCES products(id) | 商品ID |
| quantity | INT | NOT NULL DEFAULT 1 | 数量 |
| price | DECIMAL(10,2) | NOT NULL | 单价 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**设计理由：**
- 支持一个订单包含多个商品
- 记录商品当时的价格，避免价格变动影响订单
- 与 `orders` 和 `products` 表建立外键关系

**索引设计：**
- 主键 `id` 自动创建索引
- `order_id` 创建普通索引，加速查询订单的所有商品
- `product_id` 创建普通索引，便于关联商品信息

### 7. chat_messages 表（消息表）

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 消息ID |
| sender_id | INT | REFERENCES users(id) | 发送者ID |
| receiver_id | INT | REFERENCES users(id) | 接收者ID |
| content | TEXT | | 消息内容 |
| image_url | VARCHAR(255) | | 图片URL |
| status | ENUM('sent', 'delivered', 'read') | DEFAULT 'sent' | 消息状态 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

**设计理由：**
- 支持文字消息和图片消息
- `status` 字段跟踪消息状态，支持已发送、已送达、已读
- 与 `users` 表建立外键关系，记录发送者和接收者

**索引设计：**
- 主键 `id` 自动创建索引
- `sender_id` 和 `receiver_id` 创建普通索引，加速查询用户的消息
- `created_at` 创建普通索引，支持按时间排序消息

### 8. reviews 表（评价表）

| 字段名 | 数据类型 | 约束 | 描述 |
|-------|---------|------|------|
| id | INT | PRIMARY KEY AUTO_INCREMENT | 评价ID |
| order_id | INT | REFERENCES orders(id) UNIQUE | 订单ID |
| buyer_id | INT | REFERENCES users(id) | 买家ID |
| seller_id | INT | REFERENCES users(id) | 卖家ID |
| rating | INT | NOT NULL | 评分（1-5） |
| comment | TEXT | | 评价内容 |
| created_at | TIMESTAMP | DEFAULT CURRENT_TIMESTAMP | 创建时间 |

**设计理由：**
- 每个订单只能有一个评价
- 记录买家对卖家的评价
- 与 `orders` 和 `users` 表建立外键关系

**索引设计：**
- 主键 `id` 自动创建索引
- `order_id` 创建唯一索引，确保每个订单只有一个评价
- `buyer_id` 和 `seller_id` 创建普通索引，加速查询用户的评价

## 数据库约束和关系

1. **外键约束：**
   - `products.category_id` -> `categories.id`
   - `products.user_id` -> `users.id`
   - `images.product_id` -> `products.id`
   - `orders.buyer_id` -> `users.id`
   - `orders.seller_id` -> `users.id`
   - `order_items.order_id` -> `orders.id`
   - `order_items.product_id` -> `products.id`
   - `chat_messages.sender_id` -> `users.id`
   - `chat_messages.receiver_id` -> `users.id`
   - `reviews.order_id` -> `orders.id`
   - `reviews.buyer_id` -> `users.id`
   - `reviews.seller_id` -> `users.id`

2. **级联操作：**
   - 删除用户时，关联的商品、订单、消息等应根据业务逻辑处理
   - 删除商品时，关联的图片应级联删除
   - 删除订单时，关联的订单项和评价应级联删除

## 状态字段设计说明

1. **用户状态 (users.status):**
   - `active`: 正常活跃用户
   - `banned`: 被封禁用户

2. **商品状态 (products.status):**
   - `available`: 在售状态
   - `sold`: 已售出
   - `removed`: 已下架

3. **订单状态 (orders.status):**
   - `pending`: 待付款
   - `paid`: 已付款
   - `shipped`: 已发货
   - `completed`: 已完成
   - `cancelled`: 已取消
   - `refunded`: 已退款

4. **消息状态 (chat_messages.status):**
   - `sent`: 已发送
   - `delivered`: 已送达
   - `read`: 已读

## 索引设计说明

1. **主键索引：** 所有表的 `id` 字段作为主键，自动创建索引
2. **唯一索引：** 确保 `username`、`phone`、`category.name` 等字段的唯一性
3. **外键索引：** 为所有外键字段创建索引，加速关联查询
4. **状态索引：** 为 `status` 字段创建索引，便于状态筛选
5. **时间索引：** 为 `created_at` 字段创建索引，支持时间排序和范围查询
6. **业务索引：** 根据具体业务需求，为常用查询字段创建索引

## 数据类型选择说明

1. **整数类型：** 使用 `INT` 作为主键和外键，满足ID需求
2. **字符串类型：** 根据实际需求选择 `VARCHAR` 长度，避免过度浪费空间
3. **小数类型：** 使用 `DECIMAL(10,2)` 存储价格，确保精度
4. **时间类型：** 使用 `TIMESTAMP` 存储时间，自动更新
5. **枚举类型：** 使用 `ENUM` 存储状态字段，提高数据一致性
6. **文本类型：** 使用 `TEXT` 存储较长的描述和内容

## 性能优化考虑

1. **合理分表：** 对于消息表等可能快速增长的表，考虑分表策略
2. **缓存策略：** 对热门商品和分类使用缓存，减少数据库查询
3. **查询优化：** 使用索引覆盖查询，避免全表扫描
4. **事务管理：** 对订单和支付等关键操作使用事务，确保数据一致性
5. **定期维护：** 定期优化表结构，重建索引，清理过期数据

## 安全考虑

1. **密码存储：** 使用哈希算法存储密码，避免明文存储
2. **SQL注入：** 使用参数化查询，避免SQL注入攻击
3. **权限控制：** 为不同角色设置合适的数据库权限
4. **数据备份：** 定期备份数据库，确保数据安全
5. **敏感信息：** 对敏感信息进行加密存储

## 扩展性考虑

1. **预留字段：** 为可能的业务扩展预留字段
2. **模块化设计：** 采用模块化设计，便于功能扩展
3. **兼容性：** 考虑数据库版本兼容性，便于后续升级
4. **数据迁移：** 设计合理的数据迁移方案，支持系统升级

此数据库设计基于用户故事地图中的MVP需求，涵盖了核心功能所需的所有表结构和关系。通过合理的字段设计、索引优化和安全考虑，确保系统的性能、可靠性和可扩展性。
